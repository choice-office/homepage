# Landing Start — 자동 모드 라우터

랜딩페이지/홈페이지 요청을 받으면 **모드를 직접 지정하지 않아도** 이 스킬이 자동으로 상황을 판단하고 적절한 OMC 모드를 선택한다.

## 사용법

```
랜딩페이지 만들어줘 → 자동으로 deep-interview 또는 autopilot 선택
국밥집 홈페이지, 레퍼런스: [URL] → 자동으로 autopilot 선택
이 섹션들 다 만들어줘: hero, stats, gallery, faq → 자동으로 team 선택
```

---

## 자동 라우팅 로직

다음 기준으로 모드를 자동 선택한다. **사용자에게 모드를 물어보지 말고 직접 판단해서 실행**한다.

### 1. 정보 충분도 평가

아래 항목을 체크한다:

| 항목 | 있음 | 없음 |
|------|------|------|
| 업종/업태 | +2 | 0 |
| 레퍼런스 URL | +2 | 0 |
| 원하는 섹션 목록 | +2 | 0 |
| 색상/분위기 | +1 | 0 |
| 연락처/주소 등 실제 데이터 | +1 | 0 |

**점수 0~2점**: 정보 부족 → `/deep-interview` 모드

**점수 3~4점**: 기본 정보 있음 → `/autopilot` 모드

**점수 5점 이상**: 명확한 기획 → `/autopilot` 또는 명시적 섹션 5개 이상이면 `/team 3:executor`

### 2. 작업 성격별 분기

- **새 프로젝트 전체 구축**: 점수 기반 위 로직
- **기존 섹션 수정/추가**: 바로 실행 (모드 불필요)
- **버그/에러 수정 여러 파일**: `/ultrawork`
- **한 가지 기능 완성까지**: `/ralph`
- **반드시 검증 필요한 완성**: `/ralph`

## 실행 절차

### STEP 1. 자동 판단 (항상 먼저)

```
입력 분석:
- 업종 파악
- 레퍼런스 URL 존재 여부
- 섹션 요청 수
- 실제 콘텐츠 데이터 유무
→ 점수 계산 → 모드 결정
```

### STEP 2. 모드별 실행

**deep-interview 선택 시** (점수 0~2):
"좋습니다. 홈페이지를 잘 만들기 위해 몇 가지 여쭤볼게요." 라고 말하고 아래 질문을 순서대로 확인:
1. 업종/서비스 한 줄 설명
2. 주요 고객층 (연령대, 지역 등)
3. 경쟁사나 참고하고 싶은 사이트
4. 꼭 들어가야 할 섹션 (ex: 메뉴판, 오시는길, 예약폼)
5. 브랜드 컬러나 분위기 (있으면)

충분한 정보가 모이면 `landing-kickoff` 스킬로 넘어가 실행.

**autopilot 선택 시** (점수 3~4):
`landing-kickoff` 스킬을 실행해 8단계 프로세스 진행.

**team 선택 시** (섹션 5개 이상, 점수 5+):
```
/team 3:executor "landing-kickoff 스킬 기반으로 [업종] 랜딩페이지 구현:
- 에이전트 1: Hero, Features, Stats
- 에이전트 2: Gallery, Process, Testimonials
- 에이전트 3: FAQ, CTA, Contact, Map
공통: config/site.ts 수정, page.tsx 조합"
```

### STEP 3. 완료 후 항상

1. `pnpm lint` 통과 확인
2. `pnpm build` 성공 확인
3. 주요 변경사항 요약 (한국어)
4. 커밋 메시지 제안

|------|------|----------|

| "lint 에러 다 고쳐줘" | - | ultrawork |
