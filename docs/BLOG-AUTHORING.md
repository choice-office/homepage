# 블로그 작성기능(어드민) 설계 — 작성 포맷 · UX

> 상태: **구현됨** — 관리자(`choice-admin`)에서 Tiptap 기반 작성기로 제공(이 설계를 따름). 공개 렌더/JSON-LD/데이터 모델은 `docs/BLOG-SEO.md` 참고.
> **제1원칙: AI 없음(전부 규칙/라이브러리). 그리고 비전문가가 쉽고 빠르게 — 어렵거나 복잡하면 실패다.**

## 0. 절대 기준
1. **AI 0.** slug 생성·메타 기본값·품질 체크 전부 **규칙 기반**(라이브러리/문자열 처리). LLM 호출 없음.
2. **극단적 단순함.** *최소 필수 = 제목 · 카테고리 · 본문.* 나머지는 전부 **자동 또는 선택(접힘)**. "쓰고 발행" 두 단계로 끝나야 함.
3. **구조는 강요하지 말고 "기본값"으로 유도.** 템플릿·전용 입력·가드레일로 자연스럽게.

## 1. 화면 구성 (단순 2영역)
```
┌───────────────────────────┬──────────────────┐
│  본문 에디터(WYSIWYG)      │  발행 설정(사이드) │
│  - 제목 입력(상단 큰 칸)   │  [기본] 카테고리   │
│  - 본문(시맨틱 툴바)       │       커버+alt    │
│                            │       slug(자동)  │
│                            │  ─ 접힘 ─         │
│                            │  [SEO] 검색제목/설명│
│                            │  [AEO] 요점/FAQ/출처│
│                            │  [작성자]          │
│  하단: 실시간 체크리스트    │  [임시저장][발행]  │
└───────────────────────────┴──────────────────┘
```
- **기본 섹션만 펼쳐두고 SEO/AEO/작성자는 접어둔다.** 안 건드려도 발행 가능(자동 기본값).

## 2. 본문 에디터 = "시맨틱 WYSIWYG" (네이버/velog 느낌, AI 없음)
- 추천 라이브러리: **Tiptap**(ProseMirror) — HTML 출력, 무료, 로컬. (AI 아님)
- 툴바를 **시맨틱 요소에 1:1 매핑**:
  - 제목은 **드롭다운(제목2/제목3)** 으로만 → "폰트 크게+볼드" 가짜 제목 금지(검색·AI는 진짜 `<h2>`만 인식).
  - 굵게/기울임/링크/**목록**/**인용**/**구분선**/**이미지(캡션)** 정도만. 글자색·폰트크기 등 SEO 무의미 장식은 빼서 단순화.
- **붙여넣기 정리(sanitize-on-paste)**: 네이버·워드에서 붙여넣어도 잡 스타일/클래스 제거 → 허용 태그로 정규화. (규칙 기반, AI 아님)
- 출력은 **정제된 HTML** → `posts.content`.

## 3. "빈칸 채우기" 템플릿으로 시작 (구조를 디폴트로, AI 없음)
새 글 생성 시 본문을 **스켈레톤**으로 미리 채움:
```
[요점] 이 글의 결론을 2~3문장으로 적어주세요. (검색·AI가 이 부분을 답으로 인용)
## (질문형 소제목 예: F-6 심사에서 무엇을 보나요?)
본문…
## (필요하면 소제목 추가)
```
- 고객은 placeholder만 덮어쓰면 좋은 구조 완성. SEO 지식 0이어도 됨.
- 강제 아님(지워도 됨). 단지 "쉬운 정답 경로".

## 4. 구조 요소 = 본문 밖 "전용 입력"(선택, 한 번에 추가)
JSON-LD가 안정적으로 나오도록 헷갈리기 쉬운 건 폼으로 분리. **전부 선택**:
- **요점(TL;DR)**: 한 줄~세 줄 텍스트.
- **FAQ**: "질문/답 추가" 버튼으로 0~N개. (있으면 가시 블록 + FAQPage)
- **참고 출처**: 라벨+링크 추가(하이코리아·법령 등).
- **작성자**: select(자격 포함). 기본값 = 대표 행정사.

## 5. 이미지 = 업로드 + alt (네이버 문제 동시 해결, AI 없음)
- URL 붙여넣기 대신 **파일 업로드 → Supabase Storage 재호스팅**(핫링크 깨짐 해결).
- 삽입 시 **"이미지 설명(alt)" 입력**(간단 모달). 비우면 삽입 막거나 경고. (접근성+이미지 SEO)

## 6. 자동 처리 (고객은 몰라도 됨, 전부 규칙 기반)
- **slug**: 영문 칸 비우면 `{categoryCode}-{shortId}` 자동(예 `f6-3a8c`). 또는 간단 로마자 변환 라이브러리. **AI 아님.**
- **메타 제목/설명**: 비우면 title / 본문 앞 155자 자동.
- **발행일/수정일**: 자동(now / updated_at).
- **JSON-LD·OG·면책 문구·작성자 박스**: 자동.

## 7. 실시간 체크리스트 (규칙 기반 soft 경고, AI 없음)
발행은 **항상 가능**, 권장만 표시(빨강 강제 X):
- 제목 28~60자 / 질문형·키워드 포함
- 첫 문단(요점) 있음
- H2 3개+ / 질문형 권장
- 이미지 alt 모두 채움
- FAQ 1개+ · 내부링크 1개+ · 출처 1개+
- 메타설명 있음(또는 자동값 사용)
→ 단순 문자열·DOM 검사로 구현. 점수/배지로 "좋은 글" 유도.

## 8. 편의 기능 (복잡하지 않게)
- **미리보기**: 실제 렌더(요점·FAQ·작성자 포함) 그대로.
- **임시저장(draft)/자동저장**, 발행/되돌리기.
- **한글 위주**: 영문·메타는 자동이라 영어 부담 0.

## 9. 고객 작성 시나리오 (가장 단순 경로)
1. "새 글" → 제목 입력(→ slug 자동) → 카테고리 선택.
2. [요점] 덮어쓰고 질문형 소제목에 본문 작성(붙여넣어도 자동 정리).
3. 이미지 드래그 업로드 → 설명(alt) 입력.
4. (선택) FAQ 2~3개·출처 추가 → 체크리스트 초록불.
5. 미리보기 → **발행** → `/blog` 즉시 노출.
> SEO/AEO를 전혀 몰라도 1~3만 하면 발행되고, 4를 하면 점수가 올라간다.

## 10. 구현 스택 메모 (확정 아님, AI 없는 후보)
- 에디터: Tiptap. 본문 HTML 저장.
- 저장: Supabase `posts`(스키마 `docs/BLOG.md`) + Storage(이미지). RLS: 공개 읽기 published만, 쓰기 service_role/관리자.
- 노출: 목록/상세에 `revalidate`(ISR) → 발행 즉시 반영. 렌더 코드는 `docs/BLOG-SEO.md` 그대로 재사용.
- 관리자 보호: Supabase Auth 또는 단순 보호 라우트.
- (선택, 나중) "URL에서 불러오기"는 readability+turndown 등 **라이브러리만**으로 초안 생성 → 사람이 검토(역시 AI 불필요). 단 네이버 본문 추출·이미지 재호스팅이 까다로우니 우선순위 낮음.
